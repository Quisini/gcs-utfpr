# Repositorio para o trabalho de GCS do professor Rafael

## Descricao

Este repositorio foi desenvolvido como parte da disciplina de Gerencia de Configuracao de Software (GCS).

O objetivo do projeto e aplicar conceitos de versionamento, controle de configuracao, colaboracao em equipe e gerenciamento de releases utilizando Git e GitHub.

## Tema do Projeto

O sistema desenvolvido possui como tema uma loja de departamento.

O repositorio contem classes Java que representam entidades e processos relacionados ao funcionamento de uma loja, como clientes, funcionarios, produtos, vendas e estoque.

## Objetivos

* Aplicar praticas de versionamento de software.
* Utilizar tags para identificacao de versoes.
* Registrar alteracoes por meio de commits padronizados.
* Realizar integracao de contribuicoes utilizando GitHub.
* Desenvolver um conjunto de classes relacionadas a um mesmo dominio.

## Estrutura do Repositorio

### README.md

Arquivo contendo a descricao geral do projeto e orientacoes para consulta do repositorio.

### CONTRIBUTING.md

Arquivo contendo as convencoes de contribuicao, padrao de commits e politica de versionamento.

### Arquivos Java

Classes desenvolvidas pelos integrantes para compor o sistema de loja de departamento.

## Versionamento

O projeto utiliza versionamento baseado nas atividades realizadas por cada integrante.

Cada participante e responsavel por uma versao principal identificada por uma tag especifica.

As informacoes detalhadas sobre contribuicao e releases podem ser consultadas no arquivo CONTRIBUTING.md.

## Integrantes

Cada integrante contribui com classes, documentacao ou melhorias relacionadas ao sistema, seguindo as convencoes definidas para o projeto.

## Observacoes

Este projeto possui finalidade academica e foi desenvolvido para demonstrar conceitos de Gerencia de Configuracao de Software, nao tendo como objetivo a implementacao completa de um sistema comercial.
